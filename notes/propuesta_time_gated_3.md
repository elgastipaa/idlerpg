# Time-Gated Systems for Idle / Incremental RPG

## 🧠 Resumen
Objetivo: diseñar sistemas **time-gated con decisiones**, que aumenten:
- Retención diaria (multi-sesión)
- Profundidad estratégica
- Progreso a largo plazo
- Monetización futura **sin caer en P2W ni dark patterns**

# 🔥 TOP 10 MECÁNICAS (DESARROLLO COMPLETO)

---

## 1. Forge Queue System
**Inspiración:** Warframe / Clash of Clans  
- **Fantasy:** "Estoy construyendo mi build incluso offline"
- **Loop:** Poner upgrades / crafts en cola → esperar → reclamar
- **Timer:** 5m – 2h
- **Decisión:** qué item priorizar
- **Recurso:** materiales + oro
- **Reward:** upgrades progresivos
- **Retención:** alta (check-ins frecuentes)
- **Monetización:** ✅ Safe (más slots, speed-ups cosméticos)
- **Riesgo:** bajo
- **Complejidad:** baja

---

## 2. Echo Charging
- **Fantasy:** "Cargo poder meta para el próximo run"
- **Loop:** asignás ecos → se cargan con el tiempo
- **Timer:** 1h – 12h
- **Decisión:** qué bonus potenciar
- **Recurso:** echoes
- **Reward:** buffs persistentes
- **Retención:** alta

- **Riesgo:** bajo
- **Complejidad:** baja

## 4. Boss Hunt Contracts
**Inspiración:** Monster Hunter / PoE target farming  
- **Fantasy:** "Cazo lo que necesito"

- **Retención:** muy alta
- **Monetización:** ⚠️ Medium (cuidado con vender target farming)
- **Riesgo:** trivializar loot RNG
- **Complejidad:** media

# 🧩 CONCLUSIÓN

La clave no es el timer.  
Es la **decisión antes del timer**.

Si el jugador siente:
- "Elegí bien"
- "Optimicé algo"
- "Estoy construyendo algo a futuro"

→ el sistema funciona.

Si siente:
- "Estoy esperando"

→ el sistema murió.
