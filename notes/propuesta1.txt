1. DIAGNÓSTICO

Los talentos actuales son mejoras flat sin jerarquía narrativa — el jugador no siente que está construyendo algo, solo comprando bonuses
Sin nodos escalables infinitos, el jugador de nivel 77 acumula 50 puntos sin uso — el sistema no tiene dónde poner los puntos late game
Los prereqs actuales son binarios (tenés el talento o no) sin mostrar visualmente el camino hacia adelante
Los talentos "aura" se aplican al desbloquear pero el jugador no ve el impacto inmediato — feedback débil
No hay diferencia de identidad entre un Berserker que tomó distintos caminos — todas las builds Berserker se sienten iguales
Los talentos II existen pero son el mismo nodo mejorado, no una decisión real
No hay sink de puntos para late game — el sistema no fue diseñado para nivel 77+
La lectura del árbol en mobile es difícil porque los nodos son cards con demasiado texto
No hay "builds imposibles de replicar" — cualquier jugador llega a los mismos talentos eventualmente
El costo flat (1-2 TP) hace que todos los talentos se sientan igual de importantes


2. TRES ENFOQUES
Enfoque A — Árbol por Capas con Keystones
Reglas de desbloqueo: el árbol tiene 4 capas. Para acceder a la capa siguiente necesitás haber gastado X puntos en la capa anterior (no en nodos específicos). Dentro de cada capa los nodos tienen prereqs entre sí.
Cómo evita puntos sobrantes: la capa 4 tiene nodos infinitos (softcap con diminishing returns). Siempre hay dónde poner puntos.
Cómo muestra precedencia: líneas visuales entre nodos + indicador "gastá 3 más en esta capa para desbloquear la siguiente".
Pros: muy claro para el jugador, fácil de implementar, escala naturalmente.
Contras: menos libertad de build, puede sentirse lineal.

Enfoque B — Tracks por Rol con Puentes (RECOMENDADO)
Reglas de desbloqueo: tres tracks paralelos (Daño / Sustain / Utilidad). Cada track tiene sus propios nodos. Los "puentes" son nodos especiales que requieren puntos de dos tracks distintos y dan sinergias únicas.
TRACK DAÑO:        nodo1 → nodo2 → nodo3 → keystone
TRACK SUSTAIN:     nodo1 → nodo2 → nodo3 → keystone
TRACK UTILIDAD:    nodo1 → nodo2 → nodo3 → keystone
                         ↘         ↙
                       PUENTE (sinergia única)
Cómo evita puntos sobrantes: cada track tiene 2-3 nodos infinitos al final. Los puentes cuestan muchos puntos. Las keystones son caras.
Cómo muestra precedencia: tres columnas visuales con conexiones horizontales (puentes) destacadas en color distinto.
Pros: decisiones de build reales, builds con identidad distinta, los puentes crean combinaciones memorables.
Contras: más complejo de implementar, requiere balance cuidadoso de los puentes.

Enfoque C — Nodos Clave + Infinitos
Reglas de desbloqueo: 8-10 nodos clave con prereqs fijos. Alrededor de cada nodo clave hay 2-3 nodos infinitos (scalables hasta nivel 20) que amplifican ese nodo clave.
Cómo evita puntos sobrantes: los nodos infinitos absorben todos los puntos de late game con diminishing returns.
Cómo muestra precedencia: el nodo clave está en el centro, los infinitos alrededor en gris hasta que se desbloquea el clave.
Pros: extremadamente claro, siempre hay dónde poner puntos.
Contras: puede sentirse repetitivo stackear el mismo nodo infinito.

3. RECOMENDACIÓN FINAL — Enfoque B (Tracks con Puentes)
Justificación:

Los tracks dan identidad de build inmediata — el jugador dice "soy un Berserker de Daño con toque de Sustain"
Los puentes crean las combinaciones memorables que se comparten entre jugadores
Los nodos infinitos al final de cada track resuelven el problema de puntos sobrantes
Técnicamente es un grafo dirigido — tu estructura actual de prereqs[] ya lo soporta
La UI de tres columnas es más legible en mobile que el árbol libre actual

5. ECONOMÍA Y ESCALADO
Ganancia de Talent Points:
jsfunction getTalentPointsGained(level) {
  // Niveles 1-20: 1 TP por nivel (20 TP totales)
  if (level <= 20) return 1;
  
  // Niveles 21-40: 1 TP cada 2 niveles (10 TP adicionales)
  if (level <= 40) return level % 2 === 0 ? 1 : 0;
  
  // Niveles 41-60: 1 TP cada 3 niveles (7 TP adicionales)
  if (level <= 60) return level % 3 === 0 ? 1 : 0;
  
  // Niveles 61+: 1 TP cada 5 niveles
  return level % 5 === 0 ? 1 : 0;
}

Nodos infinitos — fórmula de costo:
jsfunction getInfiniteNodeCost(currentLevel) {
  return Math.floor(2 * Math.pow(1.3, currentLevel));
  // Nivel 1: 2 TP
  // Nivel 5: 7 TP
  // Nivel 10: 27 TP
  // Nivel 15: 100 TP
  // Nivel 20: 374 TP
}
Soft cap de nodos infinitos:
jsfunction getInfiniteNodeEffect(baseEffect, currentLevel) {
  return baseEffect / (1 + currentLevel * 0.15);
  // Nivel 1: 100% del efecto base
  // Nivel 5: 57% del efecto base
  // Nivel 10: 40% del efecto base
  // Nivel 20: 25% del efecto base
}

Estrategia de rollout:
js// En stateInitializer.js — al cargar el save
function migrateTalents(savedState) {
  if (savedState.talentSystemVersion >= 2) return savedState;

  const oldTalents = savedState.player.unlockedTalents || [];
  let refundedPoints = 0;
  const newTalentLevels = {};

  for (const oldId of oldTalents) {
    const migration = TALENT_MIGRATION_MAP[oldId];
    if (!migration) {
      // Talento eliminado — refund completo
      refundedPoints += 2;
      continue;
    }
    const { newId, refundExtra } = migration;
    newTalentLevels[newId] = (newTalentLevels[newId] || 0) + 1;
    refundedPoints += refundExtra;
  }

  return {
    ...savedState,
    talentSystemVersion: 2,
    player: {
      ...savedState.player,
      talentLevels: newTalentLevels,       // nuevo formato: { nodeId: nivel }
      talentPoints: (savedState.player.talentPoints || 0) + refundedPoints,
      unlockedTalents: Object.keys(newTalentLevels), // compatibilidad
    },
  };
}

Pseudocódigo de funciones:
jsfunction canUnlockNode(state, nodeId) {
  const node     = TALENT_NODES.find(n => n.id === nodeId);
  const levels   = state.player.talentLevels || {};
  const currentLevel = levels[nodeId] || 0;

  if (currentLevel >= node.maxLevel) return false;

  const cost = getNodeUpgradeCost(state, nodeId);
  if (state.player.talentPoints < cost) return false;

  // Verificar prereqs
  for (const prereqId of (node.prereqs || [])) {
    const reqLevel = node.prereqLevels?.[prereqId] || 1;
    if ((levels[prereqId] || 0) < reqLevel) return false;
  }

  // Verificar clase y spec
  if (node.classId && node.classId !== state.player.class) return false;
  if (node.specId  && node.specId  !== state.player.specialization) return false;

  return true;
}

function getNodeUpgradeCost(state, nodeId) {
  const node         = TALENT_NODES.find(n => n.id === nodeId);
  const currentLevel = (state.player.talentLevels || {})[nodeId] || 0;
  return Math.floor(eval(node.costFormula.replace("nivel", currentLevel)));
}

function applyNodeUpgrade(state, nodeId) {
  if (!canUnlockNode(state, nodeId)) return state;

  const cost         = getNodeUpgradeCost(state, nodeId);
  const currentLevel = (state.player.talentLevels || {})[nodeId] || 0;
  const newLevel     = currentLevel + 1;

  let p = {
    ...state.player,
    talentPoints: state.player.talentPoints - cost,
    talentLevels: { ...(state.player.talentLevels || {}), [nodeId]: newLevel },
    unlockedTalents: [...new Set([...(state.player.unlockedTalents || []), nodeId])],
  };

  // Aplicar efectos de aura/passive inmediatamente
  p = applyTalentEffect(p, nodeId, newLevel);
  return { ...state, player: refreshStats(p) };
}

function getAvailableNodes(state) {
  return TALENT_NODES.filter(node => {
    const currentLevel = (state.player.talentLevels || {})[node.id] || 0;
    if (currentLevel >= node.maxLevel) return false;
    return canUnlockNode(state, node.id);
  });
}

TP totales aproximados: 35-40
Trade-offs: muere si el boss tiene mecánica de burst. Destroza tiers medios pero en tiers altos necesita micro-gestión de HP. El Deseo de Muerte es mejor cuanto más bajo esté el HP — genera tensión constante.

Build 2 — Bruiser Sostenible
Concepto: daño moderado, altísima supervivencia, farmea indefinidamente.
Ruta de talentos (orden):

9. w_sus_inf (x5+)        → Resistencia Eterna

TP totales aproximados: 35-42
Trade-offs: DPS bajo — tarda más en matar. Ideal para farmear tiers donde el glass cannon moriría. El Golpe de Escudo hace que la defensa tenga doble valor (sobrevivir + pegar).
