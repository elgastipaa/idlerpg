13 Do / Don't Guide

✓ DO

Usar oro como estructura y jerarquía visual
Usar color semánticamente (verde = vida/éxito, rojo = daño/error, violeta = arcano/XP)
Mantener puntos ornamentales en separadores (diamante dorado)
Amplificar alto contraste y legibilidad
Usar tipografía consistente (Cinzel display, Barlow UI)
Diseñar primero para mobile, escalar a tablet/desktop
Icon frames circulares para talentos y nodos
Un CTA principal dorado por pantalla
Números protagonistas, labels secundarios muted
Rareza por borde y glow, no por color de fondo completo
Estados disabled/loading/success/error en todos los controles

✗ DON'T

No usar botones planos tipo SaaS sin borde ni profundidad
No usar verde como CTA principal (es semántico, no decorativo)
No usar degradados modernos saturados ni UI clara o blanca
No mezclar groovers de borde por pantalla (normalizar al UI Kit)
No sobrecargar un prototipo con brillos decorativos constantes
No inventar colores locales sin rol global definido
No copiar una captura vieja por encima del UI Kit
No usar separadores grises planos (usar ornamental dorado)
No usar emojis ni iconos cartoon en UI final
No pintar toda la card con el color de rareza
No cambiar layout entre estados (pressed/loading/disabled)
No usar dorado fuerte en textos descriptivos largos
BUILD IN DARKNESS, RISE IN LIGHT
