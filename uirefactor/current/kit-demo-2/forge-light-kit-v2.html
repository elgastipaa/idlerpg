Do / Don't Guide

✓ DO

Usar oro como estructura y jerarquía visual
Usar color semánticamente — verde vida/éxito, rojo daño/error
Separadores ornamentales con diamante dorado
Cinzel para display, Barlow Condensed para UI
Un solo CTA dorado por pantalla
Icon frames circulares con SVG rings para talentos y items
Rareza por borde y glow, no color de fondo completo
Números protagonistas, labels muted
Botones chamfered — identidad táctil real
Estados disabled/loading/success/error en todos los controles
Textura de ruido sutil sobre el fondo

✗ DON'T

No usar botones planos sin borde ni profundidad
No usar verde como CTA principal
No usar degradados modernos saturados ni UI clara
No mezclar border-radius redondeados con chamfered
No saturar con glows constantes — solo en estados activos
No inventar colores locales sin rol global definido
No copiar capturas viejas por encima del UI Kit
No usar separadores grises planos
No usar emojis ni iconos cartoon en UI final
No pintar toda la card con el color de rareza
No usar icon frames cuadrados donde corresponde circular
